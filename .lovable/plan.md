# Fix Improvement Feedback (Bullet 5) — band mislabeling & unrealistic ranges

## Problems observed

1. **English, 81.0% (A band)** → feedback says *"high B band — roughly 1% to 5% away"*. The student is already in A, but the sub-tier ladder labels 81 as "high B" because the ladder's `high B = 82` boundary disagrees with the actual letter scale (`A` starts at 81).
2. **Low D student** → feedback says *"low D band — roughly 3% to 19% away"*. They're already in low D, and the 19% upper bound is generated by `cushion = ceil(sd/2)` blowing up with high variance.
3. The "between bands" detector (±2%) only triggers near sub-tier mins, so a student right on a real letter boundary (e.g. 81, 71) gets labelled as the *lower* letter.

## Root causes (in `src/components/grade/AcademicFeedback.tsx`)

- `NEXT_TIER_LADDER` thresholds (low/mid/high A, B, C…) don't align with `REPORT_SCALE` letter mins (A 81, B 71, C 61, D 51…). A score of 81 falls into `high B (82)` per the ladder even though the letter scale says A.
- `nextTierGoal()` picks `next = ladder.find(b.min > pct)`. When the student is at the bottom of a band (e.g. 55 = low D min), the "next" is `mid D` 4 pts away — fine — but the **current band phrase** still says "low D", so the suggestion reads as if the student must climb into the band they're already in.
- `cushion = max(2, ceil(sd/2))` is uncapped → with sd≈34 it produces "+17", giving a 19% upper bound that's meaningless.
- No floor on `low` when the student is *already at or above* the next sub-tier threshold.

## Fix plan

### 1. Align the ladder with the real letter scale
Rebuild `NEXT_TIER_LADDER` so each letter's `low` tier == that letter's `REPORT_SCALE.min`:

```
E:  low 41, mid 45, high 48
D:  low 51, mid 55, high 58
C:  low 61, mid 65, high 68
B:  low 71, mid 75, high 78
A:  low 81, mid 85, high 88
A*: 91
```

This guarantees `currentBandPhrase(81)` resolves to "low A", not "high B".

### 2. Make `currentBandPhrase` letter-aware
First locate the **letter** via `REPORT_SCALE` (single source of truth), then pick the sub-tier within that letter. Boundary phrase ("between higher X and lower Y") only fires when the score is within ±1.5% of a **letter** boundary, not a sub-tier boundary.

### 3. Rework `nextTierGoal` to produce realistic, attainable goals
- Compute `gap = next.min - pct`. If `gap < 1`, round to 1.
- **Cap cushion**: `cushion = clamp(ceil(sd/3), 1, 4)` — never wider than 4 pts.
- Cap the entire upper bound so `high ≤ gap + 4` and `high ≤ (next-next.min - pct)` (never overshoot the tier *after* the goal).
- If the student is **already within 1%** of the next sub-tier, skip "X% away" wording and switch to a micro-goal: *"You're on the cusp of {nextLabel} — one strong task pushes you over."*
- If `pct ≥ 89` (already in high A), recommend stretch toward A* with "~{91-pct}% to clear A* threshold".

### 4. Use the student's own task statistics for an interim, attainable target
Pass the subject's `done[]` tasks into `nextTierGoal`. Compute:
- `bestPct` = highest single task %.
- `recentAvg` = mean of last 3 tasks.

If `bestPct > pct + 2`, append a personalised line: *"Your highest task this term was {bestPct}% — repeating that level across remaining work would lift the average toward {projected}%."*
If `recentAvg > pct`, mention upward momentum; if lower, flag the dip instead of suggesting an unrealistic jump.

### 5. Handle the "already past the suggested tier" edge case
If `pct ≥ currentLetterBand.min` AND the bullet would otherwise say "bring your grade up into the {sameLetter}", swap to the **next letter up** (e.g. low D student → goal is mid D, not low D). Concretely: filter ladder entries to `b.min > pct + 0.5` before picking `next`.

### 6. Update unit tests
In `src/components/grade/AcademicFeedback.regression.test.ts` add cases:
- 81.0 → phrase contains "low A band", goal is "mid A".
- 55.0 (low D, sd 30) → phrase contains "low D", goal is "mid D", upper bound ≤ 8.
- 80.6 → phrase contains "high B", goal is "low A", range ≤ 5.
- 89.5 → goal targets A*, gap ≈ 2.
- 70.5 (boundary) → "between higher high C band and lower low B band".
